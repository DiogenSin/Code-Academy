Paveldėjimas

Paveldėti stilius gali tik vaikinis elementas iš tėvinio elemento.
Paveldinimi stiliai yra tie stiliai, kurie keičia elemento išvaizdą, bet ne būseną.
Pvz.: color, font-size, font-family ir t.t.
Nepaveldimi stiliai yra tie, kurie keičia elemento būseną.
Pvz.: width, height, display, border, position ir t.t.

Numatytieji stiliai

Numatytieji stiliai yra tag`ams automatiškai naršyklės suteikti stiliai. Tarp skirtingų naršyklių, numatytieji stiliai gali minimalistiškai keistis ir šitaip sugadinti svetainės vaizdą skirtingų naršyklių vartotojams.
Tam, kad nebūtų problemų kuriant dizainą skirtingoms naršyklėms, yra naudojami Reset arba Normalize CSS`ai.
Reset - resetina visus stilius (ištrina)
Normalize - suvienodina visus stiloius tarp naršyklių ir panaikina dalį kenkėjiškų stilių.

Reset ir Normalize

Web naršyklės pritaiko savo numatytuosius stilių nustatymus, kurie dažnai gali iškraipyti vaizdą ir sugadinti svetainės dizainą. Dėl šios priežasties šie numatytieji stiliaus nustatytmai yra arba reset`inami arba normalizuojami.

Reset nuoroda: Meyerweb
Normalize nuoroda: Necolas

CSS prioritetai


  1. CSS points
    1. Inline style - 1000 taškai
    2. ID - 100 taškai
    3. Class - 10 taškai
    4. Element - 1 taškai
  2. CSS order (in CSS code)
  3. Inheritence
  4. !Important - naudoti tik paskutiniu atveju.
    Important ignoruoja visus prioritetus ir suteikia jam priskirtą stilių, kurio niekas negali pakeisti išskyrus kitą Important su didesniu pioritetu.

Google

Esant dviem selektoriams, kurie nusako to paties elemento vienodus stilius yra atsižvelgiama į tų selector`ių prioritetus. Aukščiau esantys prioritetai (turintys aukštesnį balą) turi pirmenybę.
Esant dviem vienodo prieritetų balo selektoriams, keičiant tą patį elemento stilių yra atsižvelgiama į CSS eiliškumą (CSS order). Kuris aprašytas vėliau, tas ir bus pritaikomas.

Icons

Mažas paveiksliukas.
Ikonos neturi joms priskirto tag`o, todėl norint jas panaudoti, reikia naudoti i arba span (naudojamas išskirti/įdėti kažkį content`ą ir jį manipuliuoti) tag`ą.
I tag`as naudojamas dėl sviejų paprastų priežasčių: 1. pasirašo kur kas greičiau nei span ir užima mažiau vietos 2. i - ikona, lengviau prisiminti.
Ikonos stilizuojamos kaip ir tekstas. Joms nėra priskiriamas width/height, color, tačiau naudojamas font-size, color ir t. t..
